# simulation-study-template

A reproducible project template for simulation-heavy research in ecological disease dynamics. Built to support computational literacy and good research practices from day one.

---

## 👋 Welcome

If you are using this template, you are already making a good decision. This structure is designed to keep your project organized, your analysis reproducible, and your future self grateful. It is built around the reality of simulation-based research — iterative, computationally intensive, and full of decisions that are easy to forget if you do not write them down.

Take a few minutes to read through the READMEs in each folder before you start. They are short and worth it.

## 📁 Project structure
```
simulation-study-template/
├── data/
│   ├── raw/            # Original, untouched data (never edited)
│   └── processed/      # Cleaned and prepared data, output of R/ scripts
│
├── R/                  # Standalone R scripts for processing and simulations
│
├── notebooks/          # Quarto documents — your digital lab notebook
│
├── outputs/            # Simulation outputs, organized by run
│
├── figures/            # Figures generated by R scripts
│
├── manuscript/         # Everything needed to compile the paper
│
└── README.md           # You are here
```

## 🔁 The research loop

This project is designed around an iterative workflow. Here is how the pieces connect:

1. **Explore** — open a notebook in `notebooks/`, look at your data, design your simulation, and document your reasoning
2. **Run** — write or update a script in `R/`, run it locally or submit to HPC
3. **Collect** — simulation outputs land in `outputs/`, organized by run
4. **Visualize** — load outputs in a notebook, explore results, then write a figure script in `R/`
5. **Finalize** — polished figures go to `figures/`, then into the manuscript in `manuscript/`
6. **Repeat** — go back to step 1 as needed

The notebooks are the connective tissue of this loop. They are where you think out loud, and where the reasoning behind every script and every run is documented.
